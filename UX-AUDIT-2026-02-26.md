# Website UX Audit — User Navigation & Content Clarity

**Date:** 2026-02-26 16:03 PST  
**Reviewer:** Chief  
**Focus:** Navigation clarity, content discovery, user understanding of routes/pages

---

## Executive Summary

**The problem:** Site has clean design but poor **information architecture**. Users don't understand what's on each page, where content lives, or how to navigate to specific pieces.

**Key issues:**
1. **Homepage post links broken** — all go to `/ideas` instead of individual posts
2. **Learn module directory confusing** — file explorer metaphor doesn't communicate value
3. **No page descriptions** — users arrive at pages without knowing what they'll find
4. **Navigation lacks context** — nav bar shows labels but not what those sections contain
5. **Content buried** — 9 blog posts exist but homepage only shows 4, with no clear path to others

---

## Issue Breakdown

### 1. Homepage — Broken Links & Poor Signposting

**Problem:** Latest posts section shows 4 posts but ALL link to `/ideas` instead of individual articles.

**Current code:**
```tsx
<PostRow
  title="The Bespoke AI Model: Why We Build Inside Your Walls"
  category="business"
  readTime="10 min"
  date="Feb 25"
/>
```

These rows have no href — they implicitly link to parent `/ideas`.

**Fix:**
```tsx
<Link href="/ideas/bespoke-ai-model">
  <PostRow
    title="The Bespoke AI Model: Why We Build Inside Your Walls"
    category="business"
    readTime="10 min"
    date="Feb 25"
  />
</Link>
```

**Better approach:**
Pull posts dynamically from `getAllPosts()` and generate links from slugs:

```tsx
const latestPosts = getAllPosts().slice(0, 4);

{latestPosts.map(post => (
  <Link key={post.slug} href={`/ideas/${post.slug}`}>
    <PostRow
      title={post.frontmatter.title}
      category={post.frontmatter.category}
      readTime={post.frontmatter.readTime}
      date={formatDate(post.frontmatter.date)}
    />
  </Link>
))}
```

**Impact:** Users can't actually read the articles you're promoting.

### 2. Learn Page — Directory Feature Confusion

**Problem:** The file explorer metaphor (`LearnExplorer`) is clever but **doesn't communicate what users will get**.

**Current UX:**
- "Step 1: expand a folder. Step 2: select a file."
- Shows modules as folders with files like `README.md`, `request.json`, `response.json`
- Only 1 of 6 modules marked `available: true`
- 5 modules show "Coming soon" with no content

**User confusion:**
- "What's in these files?" (unclear value prop)
- "Why is this a file browser?" (metaphor without payoff)
- "Where's the actual learning content?" (not obvious)
- "Only 1 module available?" (feels incomplete)

**Root cause:** The file browser UI is borrowed from developer tools (VS Code aesthetic) but your audience is **healthcare leaders, not developers**. The metaphor creates cognitive load without payoff.

**Better approach:**

**Option A: Card-based module grid (recommended)**
```
┌─────────────────────────────────────┐
│ What Is an LLM?                     │
│ 8 min · Beginner                    │
│ ───────────────────────────────────│
│ The world's best autocomplete —     │
│ how language models predict the     │
│ next token.                         │
│                                     │
│ [Start Module →]                    │
└─────────────────────────────────────┘
```

Each card shows:
- Title
- Duration + difficulty
- Description
- Availability status
- CTA button

**Option B: Keep file browser but add context**
- Above the explorer: "Each module is a self-contained lesson with examples"
- Add hover tooltips: "README.md = lesson overview"
- Show preview pane by default with welcoming content
- Replace "Step 1/Step 2" with "Browse modules → Select files to preview"

**Option C: Hybrid — tabs within each module**
Once a module is selected, show tabs:
- **Overview** (README content)
- **Examples** (request/response JSONs)
- **Practice** (interactive exercises)

### 7. Learn Page — Tab Navigation Confusion

**Problem:** 4 tabs with unclear purposes.

**Current tabs:**
1. **Overview** — "Understand the map, then choose a learning path"
2. **Guided Sim** — "Watch realistic LLM/API traces step-by-step"
3. **File Explorer** — "Inspect the source files behind each module"
4. **Deep Dives** — "Explore longer, domain-specific investigations"

**User confusion:**
- "What's the difference between Overview and Guided Sim?"
- "Why would I want to see source files?" (developer framing)
- "What's actually in Deep Dives?" (no content yet)

**Better structure:**

**Option A: Simplify to 2 tabs**
- **Modules** (all learning content)
- **Lab** (WebMCP interactive demo)

**Option B: Reframe tabs for clarity**
1. **Start Here** (orientation + first module)
2. **All Modules** (grid of available lessons)
3. **WebMCP Lab** (interactive demo)

**Option C: Remove tabs entirely**
Just show:
- Hero text: "Interactive modules that demystify AI"
- Grid of modules (available + coming soon)
- Separate CTA for WebMCP Lab at bottom

I recommend **Option C** — tabs add complexity without clear benefit.

## UX Principles to Apply

### 1. Clarity Over Cleverness
The file browser metaphor is clever but confusing. Users want to know:
- "What will I learn?"
- "How long will it take?"
- "Is this for me?"

Not: "What's in this JSON file?"

### 2. Content Discoverability
If 9 blog posts exist, surface all 9. Don't bury content behind navigation.

### 3. Progressive Disclosure
Show users what's available NOW, not what's "coming soon". Empty states create negative impression.

### 4. Call to Action
Every page should have clear next steps:
- Work → "View demo" / "Read case study"
- Ideas → "Read this post" / "Browse all posts"
- Learn → "Start this module"
- Contact → "Send message" (that actually works)

### 5. Trust Through Transparency
If only 1 learning module is complete, either:
- Show only that one (hide the rest)
- OR frame as "Module 1 available now, more coming soon"

Don't show 5 "Coming soon" placeholders — feels incomplete.
