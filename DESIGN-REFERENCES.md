## Claude Blattman (https://claudeblattman.com)

**Added:** 2026-02-26  
**Reason:** Clean academic aesthetic, excellent content hierarchy

### What We Like

**Visual Design:**
- Minimal, text-focused layout
- Clean typography
- Professional but not corporate
- High readability, low distraction

**Content Structure:**
- Clear hierarchy: Essentials → Setup → Workflows → Advanced
- Each section has descriptive preview text
- "Where to Start" orientation for new visitors
- Callout boxes for key content sections

**Navigation:**
- Simple top-level navigation
- Section anchors (¶) for deep linking
- Footer CTAs (GitHub star, feedback email)

**Tone:**
- Personal but authoritative (Chris Blattman, UChicago professor, using Claude Code)
- "I've never coded in my life, so if I can do this you can" - accessible expert
- Practical, no fluff

### Patterns to Consider

1. **Landing page as content hub** - not just links, but oriented guidance
2. **Progressive disclosure** - essentials first, advanced later
3. **Real examples** - "Tax Season Case Study" shows practical applications
4. **Continuous improvement section** - meta-content about the system itself
5. **Clear CTAs** - Star on GitHub, send feedback (low-friction engagement)

### How It Relates to Our Site

Both sites target professionals learning to use AI tools effectively. Claude Blattman focuses on Claude Code workflows; ours will cover:
- Business use cases for AI (bespoke consulting model)
- Technical deep-dives (WebMCP, agent architecture)
- Interactive learning modules

**Tone alignment:** Accessible expert. Blake has domain expertise (healthcare BD, data architecture) + learning software dev. Similar "if I can build this, you can" energy.
