# Bespoke AI Services Business Framework

## Summary

Defined the full business model for Blake's venture: delivering bespoke AI solutions to healthcare organizations, deployed inside their own Azure environments. Not SaaS — each deployment is a configured instance of proven infrastructure.

## Core Thesis

The bottleneck was never code — it's knowing what to build, having the data expertise, and understanding the domain. AI is the accelerant, not the product.

## Key Concepts

### The Paradigm Shift
- **Old model:** Build product → host it → sell subscriptions → 6-12 month sales cycles, BAAs, security reviews
- **New model:** Build inside the client's walls → their Azure tenant, their OpenAI instance, data never leaves → eliminates compliance gatekeeping
- Speed is a feature, not a shortcut — demonstrates deep expertise
- Value shifts from software to intelligence

### What We Sell (5 Layers)
1. **Domain expertise + data architecture** — Claims, CMS datasets, provider networks, entity resolution
2. **The blueprint, not the bricks** — Knowing what to build from years in healthcare BD
3. **Configuration as product** — Reusable component library, schemas, pipeline templates
4. **External data & intelligence layer (add-on)** — CMS pipelines, web crawls, Trilliant Health, third-party data
5. **White-glove upsell** — Bespoke dashboards, LLM processing pipelines, custom integrations

### Three Tangible Products
1. **Agent Data Catalog** — Consulting engagement to build structured knowledge repo ($15-40K, 2-4 weeks)
2. **Conversational AI Bots (Teams/Web)** — Standalone purpose-built bots ($10-25K per bot, 1-2 weeks)
3. **Dashboards & Bespoke Applications** — Azure Container Apps ($15-50K, 1-3 weeks)

### Pricing
- Typical first engagement: ~$55K setup + $3-5K/mo recurring
- Price on value, not time to build
- Client alternative: $270K+/yr in hires + 6 months

### The Moat (5 Layers)
1. Domain knowledge
2. Data architecture (built and proven)
3. Reusable component library (compounds with each deployment)
4. Speed (days vs. months)
5. Ongoing intelligence (subscription)

### Why Better Than SaaS for Healthcare
- No BAA for data transit
- No vendor risk assessment for hosting
- IT can inspect everything
- Client controls the AI instance
- Decommissioning is simple
